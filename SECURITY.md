# Security Policy

## Supported Versions

The desktop app is in pre-release V1 development. Security fixes should target the current `main` branch until versioned releases exist.

## Reporting A Vulnerability

Do not open a public issue for vulnerabilities involving local privacy, unsafe audio control, installer behavior, or unintended data exposure.

Contact the maintainer privately first. If no private channel is available yet, open a GitHub issue with a minimal description and no sensitive details, then request a private contact path.

## Security And Privacy Principles

- No telemetry by default.
- No account requirement.
- No audio capture storage.
- No upload of logs or audio data.
- Local logs must avoid sensitive content.
- Changes affecting privacy must update documentation and changelog.
