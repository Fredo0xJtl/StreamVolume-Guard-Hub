# StreamVolume Guard Hub v0.1.0-alpha.1

Status: alpha tester package, not a stable V1.

This note is the source text to use if a GitHub pre-release is created later.
Do not create a tag or GitHub release unless that is explicitly requested.

## What This Alpha Tests

- Windows desktop mixer control through `WindowsSessionVolume`.
- Browser extension observation and `BrowserGain` when the site/browser exposes a usable signal.
- Local bridge on `127.0.0.1:47841`.
- Honest source classification with `origin`, `controlSurface`, `status`, and `isControllable`.
- One-shot automatic volume correction to avoid continuous fader movement.
- Manual target profiles: `Calme` around 40%, `Standard` around 70%, `Fort` around 100%.
- Panic, exclusions, local logs, and copyable tester reports.
- Reproducible Windows tester package generated by `tools/package-tester.ps1`.

## Important Limits

- No custom audio driver.
- No account, cloud sync, telemetry, or automatic log upload.
- No OBS meter or scene reading in this alpha. OBS is a manual visual validation path.
- A browser can still appear as one Windows mixer source. If multiple tabs play at the same time and no usable `BrowserGain` source exists, the Windows fallback moves the whole browser.
- `BrowserGain` is valid only when the extension can measure useful signal and reaches a clear calibration state such as `locked`.
- `ObserveOnly`, `Unknown`, `skipped`, or `no-signal` means the source is visible but not directly controlled by the extension.
- YouTube and TikTok may fall back to global browser volume when direct `BrowserGain` is unavailable.

## Manual Test Order

Test one source at a time first:

1. YouTube in browser.
2. TikTok in browser.
3. Spotify Web or Deezer Web.
4. VLC or local player.
5. Discord.
6. Spotify desktop if available.
7. OBS with meters visible, manually observed.
8. Combined browser plus desktop-app scenarios.

For every source, copy the app report after the test and confirm:

- `origin` is visible.
- `controlSurface` is visible.
- `status` is understandable.
- `isControllable` matches the actual control surface.
- Non-controllable sources are not described as controllable.

## Known Follow-Up

- Real alpha logs still decide whether YouTube/TikTok/Spotify Web are mostly `BrowserGain`, `ObserveOnly`, or Windows fallback in the user's browser setup.
- A local diagnostic export would make support easier than copying reports manually.
- OBS integration remains research after the desktop/browser alpha proves the workflow.
