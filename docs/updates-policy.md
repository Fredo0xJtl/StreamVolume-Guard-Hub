# Updates Policy

Every meaningful change should update project context while the change is fresh.

Update `CHANGELOG.md` when:

- behavior changes;
- UI text or workflow changes;
- supported setup changes;
- tests or build commands change;
- known limitations change;
- privacy/security posture changes.

Update `README.md` when:

- install, build, or run commands change;
- product promise changes;
- repository structure changes;
- public positioning changes.

Update `docs/desktop-v1-cahier-des-charges.md` only when the product decision itself changes, not for every implementation detail.

Update `docs/product-next-plan.md` when the next implementation step, delivery order, or source-of-truth plan changes.
